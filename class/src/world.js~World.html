    public component(name: string, componentClass: function): string source

    Registers a component type to the world. Components must be constructable. If the component has an onCreate(), it is passed the entity first, then the rest of the arguments from methods like entity.set(). Also, components can have an onRemove() method, which gets called when removing that component from an entity.

    Params:

    NameTypeAttributeDescription
    name string

    The name
    componentClass function

    The component class, must be a constructable class or function

    Return:

    string

    Registered component name on success, undefined on failure

    Example:

    world.component('myComponent', class {
    // It is highly recommended to use onCreate() over constructor(), because the component
    // will have already been added to the entity. In the constructor(), it is not safe to use
    // "entity" because it does not contain the current component while still in the constructor.
    onCreate(entity, some, args) {
        this.entity = entity
        this.some = some
        this.args = args
    }
})
// entity === the new entity object
// some === 10
// args === 500
world.entity().set('myComponent', 10, 500)

    public every(componentNames: Array, callback: Function, args: ...Object): MapIterator source

    Iterate through entities with the specified components

    Params:

    NameTypeAttributeDescription
    componentNames Array

    The component names to match entites with. This checks if the entity has ALL of the specified components, but does not check for additional components.
    callback Function

    The callback to call for each entity. Takes (...components, entity, ...args).
    args ...Object
    • optional

    Any additional arguments to pass to the callback.

    Return:

    MapIterator

    If no callback specified, then a generator to the entities themselves. Otherwise, returns undefined.

    Example:

    // Use a callback to process entities one-by-one
world.every(['comp'], comp => {comp.value = 0})
    // Get an iterator for the entities
let it = world.every(['comp'])
for (let entity of it) {...}

    public system(args: ...Object): number source

    Registers a system to the world. The order the systems get registered, is the order then run in.

    Params:

    NameTypeAttributeDescription
    args ...Object

    Both signatures are accepted: (components, systemClass, ...args) or (systemClass, ...args).

    [components]: The list of components the system will process in every(). This follows the same logic as entity.has() and world.every().

    {systemClass}: The system class to instantiate. Can contain the following methods: constructor, initialize, pre, every, post. Pre() and post() get called before and after every(), for each of the independent systems. See world.run() for an example of the call order.

    [...args]: The arguments to forward to the system's constructors.

    Return:

    number

    Unique ID of the system on success or undefined on failure

    Example:

    // Movement system
world.system(['position', 'velocity'], class {
     constructor(context) {
         // This is showing how you can optionally pass parameters to the system's constructor
         this.context = context
     }
     every(position, velocity, entity) {
         position.x += velocity.x
         position.y += velocity.y
     }
 }, context)
    // System that doesn't use every()
world.system(class {
     constructor(context) {
         this.context = context
     }
     pre() {
         // Handle events or something
     }
 }, context)
